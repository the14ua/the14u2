Chapter 1: Database Management System

Data: Raw, unorganized facts and figures without context. Example: 98, Kathmandu, Blue. On their own, they don't convey any meaning until they are processed.

Information: Organized or processed data that provides meaning or value and is useful for decision-making. Example: "The temperature today in Kathmandu is 25°C."

Features of Information

  1. Accurate: Free from error, consistent with facts.
  2. Timely: Available when needed, not outdated.
  3. Relevant: Applicable to the purpose at hand.
  4. Complete: Includes all necessary data for a decision.
  5. Well-formatted: Organized and easy to interpret.

Database: A systematically organized collection of structured data that allows for easy access, management, and updating.

Purpose of Database: To store, retrieve, manipulate, and manage data in a structured and secure way that supports decision-making and data integrity.

Database Terminologies

  • Table: Collection of data organized in rows and columns. Each table stores data for a specific entity.
  • Field: Column in a table. Represents one attribute of the data, such as 'Name' or 'Age'.
  • Record / Tuple: Row in a table. Represents a single item or instance of data, e.g., one person’s full information.
  • Object: In object-oriented databases, it’s a combination of related data and methods that operate on that data.
  • Keys: Special fields used to identify records uniquely (Primary, Foreign, Candidate Keys).

Data Dictionary: A set of information describing the contents, format, and structure of a database and the relationship between its elements. Acts like metadata about the database.

Database Management System (DBMS)

Introduction: A DBMS is software that enables users to define, manipulate, retrieve, and manage data in databases efficiently.

Objectives: To ensure data integrity, data security, concurrent access, and reduce redundancy.

Advantages: Provides backup and recovery, minimizes data redundancy, improves data sharing and security, and maintains data integrity.

Disadvantages: Can be expensive to implement and maintain, requires trained personnel, and may be complex to manage.

Types of Database Models

  • Hierarchical: Uses a tree-like structure with parent-child relationships. Fast but rigid. Example: XML documents.
  • Network: Allows multiple parent-child relationships. More flexible than hierarchical. Example: IDMS.
  • Relational: Data is stored in related tables using keys. Most widely used. Example: MySQL, PostgreSQL.
  • Entity-Relational (ER): A diagrammatic model that visually represents data and their relationships before implementation.

Integrity Constraints

  • Domain: Ensures data entered is valid and within an acceptable range/type (e.g., age must be a number between 1–120).
  • Entity: Ensures each record is uniquely identified (e.g., every student must have a unique student ID).
  • Referential: Maintains valid relationships between tables (e.g., foreign key in orders table must exist in customers table).
  • Keys: Mechanisms to enforce constraints and provide relationships (Primary Key, Foreign Key, Composite Key, etc.).

Normalization

Introduction: A database design technique that reduces data redundancy and improves data integrity by organizing tables efficiently.

Normal Forms:

  • 1NF: Removes repeating groups and ensures each field contains only atomic values.
  • 2NF: Builds on 1NF and removes partial dependencies; applies only if a table has a composite primary key.
  • 3NF: Builds on 2NF and removes transitive dependencies where non-key attributes depend on other non-key attributes.

Advantages: Better data consistency, avoids anomalies during update/insert/delete, efficient data structure.

Disadvantages: Complex queries with multiple joins, sometimes impacts performance in read-heavy systems.

Centralized vs. Distributed Database

Introduction: Centralized databases keep all data in one location; distributed databases store data across multiple physical sites.

Advantages of Centralized: Simple to manage, better data consistency, unified security control.

Disadvantages of Centralized: High risk of data loss from failure, difficult to scale, bottlenecks.

Advantages of Distributed: Improved reliability and availability, better performance across regions, scalability.

Disadvantages of Distributed: Difficult to maintain, higher complexity, synchronization and consistency challenges.

Database Security

Introduction: Database security is about protecting sensitive data stored in a database from unauthorized access, breaches, or misuse.

Challenges: Threats from hackers, accidental data leaks, SQL injection attacks, insider threats, natural disasters.

Security Measures: User authentication, role-based access control, data encryption, regular backups, firewalls, and audit trails.

Role of DBA (Database Administrator): Ensures smooth operation of databases including design, performance tuning, setting up user roles, backup and disaster recovery, and enforcing security.

Practical Topics

DDL and DML:

  • DDL (Data Definition Language): Used to define or change structure of databases or tables (e.g., CREATE, ALTER, DROP).
  • DML (Data Manipulation Language): Used to add, delete, or modify data in existing tables (e.g., INSERT, UPDATE, DELETE).

SQL Data Types (with Detailed Use-Cases):

  • CHAR: Stores fixed-length strings (e.g., country codes like 'US', 'NP'). Always uses full space regardless of content.
  • VARCHAR: Stores variable-length strings (e.g., names, emails). Saves space by using only needed characters.
  • BINARY: Fixed-length binary data (e.g., fixed-size encryption key or flag in binary).
  • VARBINARY: Variable-length binary data (e.g., storing images, audio files, or PDFs).
  • TINYBLOB: Very small binary large object (up to 255 bytes) suitable for small media files like favicons.
  • TINYTEXT: Very small text string (up to 255 characters), good for short user bios or brief notes.
  • TEXT: Large text data (up to 65,535 characters), ideal for articles, product descriptions, reviews.
  • LONGTEXT: Extra-large text (up to 4GB), used for storing logs, legal documents, eBooks.
  • ENUM: List of predefined values (e.g., ENUM('small', 'medium', 'large') for clothing size).
  • BIT: Stores bit values (0 or 1). Good for binary flags like on/off, true/false.
  • TINYINT: Small integer (-128 to 127). Common for short ranges like age or ratings.
  • BOOLEAN: Alias of TINYINT (0 = FALSE, 1 = TRUE). Used for logical conditions.
  • INTEGER (INT): Whole numbers (e.g., 100, -25). Used for IDs, quantity, counters.
  • FLOAT: Approximate floating-point numbers. Ideal for measurements like height or weight.
  • DOUBLE: More precise floating-point number than FLOAT. Ideal for scientific or statistical calculations.
  • DECIMAL: Fixed-point precision number. Perfect for financial data like prices, salaries.
  • DATE: Stores only date (e.g., '2025-06-16'). Used for birthdays, registration dates.
  • DATETIME: Stores both date and time (e.g., '2025-06-16 14:30:00'). Perfect for event logs or timestamps.

Chapter 1: DBMS Questions & Answers

1. What is Database and DBMS? List out the advantages and disadvantages of DBMS.

Database:

A Database is a systematic collection of related data stored and accessed electronically. It is designed to manage large bodies of information by organizing it into tables, records, and fields.

DBMS:

A Database Management System (DBMS) is software that interacts with end-users, applications, and the database itself to capture and analyze data. It provides tools for defining, creating, querying, updating, and administering databases.

Advantages of DBMS

  • Reduces data redundancy and inconsistency.
  • Ensures data integrity and security.
  • Provides concurrent access and crash recovery.
  • Enables centralized data management.

Disadvantages of DBMS

  • High initial cost for software and hardware.
  • Complexity of DBMS design and management.
  • Performance overhead for small applications.
  • Requires trained personnel (DBA).

3. Explain the different models of DBMS with advantages and disadvantages.

Hierarchical Model

  • Advantage: Simple representation like a tree.
  • Disadvantage: Rigid structure, poor at many-to-many.

Network Model

  • Advantage: Supports many-to-many relationships.
  • Disadvantage: Complex schema and navigation.

Relational Model

  • Advantage: Easy to use, powerful query languages (SQL).
  • Disadvantage: Can be slower for complex joins.

Object-Oriented Model

  • Advantage: Encapsulates data and behavior.
  • Disadvantage: Less mature query support.
4. What is a relational database? How is it different from other database models?

A relational database stores data in tables (relations) consisting of rows and columns. Each row represents a record, and each column represents an attribute. Relationships between tables are established using keys.

Differences:

  • Uses declarative query language (SQL).
  • Schema-flexible: tables can be joined dynamically.
  • Ensures ACID properties.
  • Tabular view simplifies understanding.
5. What is data redundancy? How does DBMS help in reducing it?

Data redundancy occurs when the same piece of data exists in multiple places. This leads to wastage of storage and inconsistency risks.

A DBMS reduces redundancy by:

  • Centralized data storage.
  • Normalization, organizing tables to eliminate duplicate data.
  • Enforcing constraints to prevent duplicate entries.

7. Who is Database Administrator (DBA)? What are the major responsibilities of DBA?

A Database Administrator (DBA) is a professional responsible for managing and maintaining database systems to ensure optimal performance, security, and availability.

Major Responsibilities:

  • Installation and configuration of DBMS software.
  • Database design and schema creation.
  • Performance tuning and optimization.
  • Backup and recovery management.
  • Security administration: user roles and permissions.
  • Monitoring and troubleshooting database issues.
8. Define normalization. Explain 1NF, 2NF and 3NF with suitable examples. Explain the normalization process with examples.

Normalization:

Process of organizing data to minimize redundancy and dependency by dividing tables and establishing relationships.

1NF (First Normal Form):

Eliminate repeating groups; ensure each cell holds atomic values.

Example:
Before: StudentCourses(StudentID, Courses)
After: StudentCourses(StudentID, Course)

2NF (Second Normal Form):

1NF + no partial dependency; every non-key attribute fully depends on the primary key.

Example:
Table: OrderDetails(OrderID, ProductID, ProductName, Quantity)
Split into:
 Orders(OrderID, Quantity)
 Products(ProductID, ProductName)

3NF (Third Normal Form):

2NF + no transitive dependency; non-key attributes depend only on the key.

Example:
Before: Employee(EmpID, DeptID, DeptName)
After:
 Employees(EmpID, DeptID)
 Departments(DeptID, DeptName)
9. Explain the terms: primary key, foreign key, and candidate key with examples.
  • Primary Key: Unique identifier for a record. Example: StudentID in Student table.
  • Foreign Key: Field in one table that references a primary key in another. Example: DeptID in Employee table references Department(DeptID).
  • Candidate Key: Set of attributes that can uniquely identify a record. Example: {Email, PhoneNumber} in a User table if both are unique.
10. What is SQL? Explain its components and common functions.

SQL (Structured Query Language) is the standard language for interacting with relational databases.

Components:

  • DDL (Data Definition Language): CREATE, ALTER, DROP.
  • DML (Data Manipulation Language): SELECT, INSERT, UPDATE, DELETE.
  • DCL (Data Control Language): GRANT, REVOKE.
  • TCL (Transaction Control Language): COMMIT, ROLLBACK, SAVEPOINT.

Common Functions:

  • Aggregate: COUNT(), SUM(), AVG(), MIN(), MAX()
  • String: CONCAT(), SUBSTRING(), LENGTH()
  • Date: NOW(), CURDATE(), DATEDIFF()

12. Define the following terms.
  • Data Dictionary: A centralized repository of metadata describing database structure, schema, tables, columns, data types, and relationships.
  • Primary Key: A field or combination of fields that uniquely identifies each record in a table.
  • Relationship: An association between two tables, typically enforced via primary and foreign keys.
  • Data Manipulation Language (DML): Subset of SQL commands (INSERT, UPDATE, DELETE, SELECT) used to retrieve and modify data.
  • Structured Query Language (SQL): Standard language for defining, manipulating, and querying relational databases.
  • Data Integrity: Ensuring accuracy, consistency, and reliability of data over its lifecycle.
  • Data Definition Language (DDL): Portion of SQL that defines database schemas (CREATE, ALTER, DROP).
  • Data Security: Measures and controls that protect data from unauthorized access, breaches, and loss.
  • Database System: An integrated collection of software (DBMS) and data that provides mechanisms for storage, retrieval, and management of databases.

Chapter 2: C- Programming Questions & Answers

1. Working with a function

a. Define function

A function is a self-contained block of code designed to perform a specific task. It allows modular programming by encapsulating logic that can be reused.

b. Syntax of functions

return_type functionName(parameter_list) {
    // function body
    statements;
    return value; // if return_type is non-void
}

c. Types of functions

  • Library functions: Predefined in C standard libraries, e.g., printf(), scanf().
  • User-defined functions: Created by programmers to perform custom tasks.

d. Components of function

  • i. Function prototype: Declaration specifying return type, name, and parameters, placed before main().
  • ii. Function call: Statement that invokes the function, passing arguments.
  • iii. Function definition: Actual implementation including body and return statement.
  • iv. Return type: Data type of value returned; void if no value is returned.
2. Categories of function with examples

i. Function with return type but no arguments

int getValue() {
    return 100;
}

ii. Function with return type and arguments

int add(int a, int b) {
    return a + b;
}

iii. Function with no return type and no arguments

void greet() {
    printf("Hello, World!\n");
}

iv. Function with no return type but with arguments

void printSum(int x, int y) {
    printf("Sum: %d\n", x + y);
}
3. Storage classes

i. Automatic

Default storage for local variables inside functions. Lifetime is within the block, storage is reclaimed after exit.

ii. External

Global variables declared outside any function. Accessible throughout the program, lifetime is entire program execution.

iii. Register

Hints compiler to store variable in CPU register for fast access. Limited in number; "register" is advisory.

iv. Static

Maintains variable value between function calls. For local static, scope is block but lifetime is program execution; for global static, restricts linkage to file.

4. Recursive function with syntax and examples

Syntax:

return_type funcName(parameters) {
    if (base_condition) {
        return result;
    } else {
        // recursive call
        return funcName(modified_parameters);
    }
}

Example (Factorial):

int factorial(int n) {
    if (n <= 1)           // base condition
        return 1;
    else
        return n * factorial(n - 1);  // recursive call
}

The function calls itself with n-1 until reaching the base case (n <= 1).

5. Demonstration of passing array to a function

In C, arrays decay to pointers when passed to functions. The function receives the base address and can access all elements.

Code Example:

#include 

void printArray(int arr[], int size) {
    for (int i = 0; i < size; i++) {
        printf("%d ", arr[i]);
    }
    printf("\n");
}

int main() {
    int data[] = {10, 20, 30, 40, 50};
    int length = sizeof(data) / sizeof(data[0]);
    printArray(data, length);   // passing array and its size
    return 0;
}

The parameter int arr[] receives the pointer to the first element. Changes inside printArray affect original array.
