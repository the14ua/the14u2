Chapter 1: Database Management System

Data: Raw, unorganized facts and figures without context. Example: 98, Kathmandu, Blue. On their own, they don't convey any meaning until they are processed.

Information: Organized or processed data that provides meaning or value and is useful for decision-making. Example: "The temperature today in Kathmandu is 25°C."

Features of Information

  1. Accurate: Free from error, consistent with facts.
  2. Timely: Available when needed, not outdated.
  3. Relevant: Applicable to the purpose at hand.
  4. Complete: Includes all necessary data for a decision.
  5. Well-formatted: Organized and easy to interpret.

Database: A systematically organized collection of structured data that allows for easy access, management, and updating.

Purpose of Database: To store, retrieve, manipulate, and manage data in a structured and secure way that supports decision-making and data integrity.

Database Terminologies

  • Table: Collection of data organized in rows and columns. Each table stores data for a specific entity.
  • Field: Column in a table. Represents one attribute of the data, such as 'Name' or 'Age'.
  • Record / Tuple: Row in a table. Represents a single item or instance of data, e.g., one person’s full information.
  • Object: In object-oriented databases, it’s a combination of related data and methods that operate on that data.
  • Keys: Special fields used to identify records uniquely (Primary, Foreign, Candidate Keys).

Data Dictionary: A set of information describing the contents, format, and structure of a database and the relationship between its elements. Acts like metadata about the database.

Database Management System (DBMS)

Introduction: A DBMS is software that enables users to define, manipulate, retrieve, and manage data in databases efficiently.

Objectives: To ensure data integrity, data security, concurrent access, and reduce redundancy.

Advantages: Provides backup and recovery, minimizes data redundancy, improves data sharing and security, and maintains data integrity.

Disadvantages: Can be expensive to implement and maintain, requires trained personnel, and may be complex to manage.

Types of Database Models

  • Hierarchical: Uses a tree-like structure with parent-child relationships. Fast but rigid. Example: XML documents.
  • Network: Allows multiple parent-child relationships. More flexible than hierarchical. Example: IDMS.
  • Relational: Data is stored in related tables using keys. Most widely used. Example: MySQL, PostgreSQL.
  • Entity-Relational (ER): A diagrammatic model that visually represents data and their relationships before implementation.

Integrity Constraints

  • Domain: Ensures data entered is valid and within an acceptable range/type (e.g., age must be a number between 1–120).
  • Entity: Ensures each record is uniquely identified (e.g., every student must have a unique student ID).
  • Referential: Maintains valid relationships between tables (e.g., foreign key in orders table must exist in customers table).
  • Keys: Mechanisms to enforce constraints and provide relationships (Primary Key, Foreign Key, Composite Key, etc.).

Normalization

Introduction: A database design technique that reduces data redundancy and improves data integrity by organizing tables efficiently.

Normal Forms:

  • 1NF: Removes repeating groups and ensures each field contains only atomic values.
  • 2NF: Builds on 1NF and removes partial dependencies; applies only if a table has a composite primary key.
  • 3NF: Builds on 2NF and removes transitive dependencies where non-key attributes depend on other non-key attributes.

Advantages: Better data consistency, avoids anomalies during update/insert/delete, efficient data structure.

Disadvantages: Complex queries with multiple joins, sometimes impacts performance in read-heavy systems.

Centralized vs. Distributed Database

Introduction: Centralized databases keep all data in one location; distributed databases store data across multiple physical sites.

Advantages of Centralized: Simple to manage, better data consistency, unified security control.

Disadvantages of Centralized: High risk of data loss from failure, difficult to scale, bottlenecks.

Advantages of Distributed: Improved reliability and availability, better performance across regions, scalability.

Disadvantages of Distributed: Difficult to maintain, higher complexity, synchronization and consistency challenges.

Database Security

Introduction: Database security is about protecting sensitive data stored in a database from unauthorized access, breaches, or misuse.

Challenges: Threats from hackers, accidental data leaks, SQL injection attacks, insider threats, natural disasters.

Security Measures: User authentication, role-based access control, data encryption, regular backups, firewalls, and audit trails.

Role of DBA (Database Administrator): Ensures smooth operation of databases including design, performance tuning, setting up user roles, backup and disaster recovery, and enforcing security.

Practical Topics

DDL and DML:

  • DDL (Data Definition Language): Used to define or change structure of databases or tables (e.g., CREATE, ALTER, DROP).
  • DML (Data Manipulation Language): Used to add, delete, or modify data in existing tables (e.g., INSERT, UPDATE, DELETE).

SQL Data Types (with Detailed Use-Cases):

  • CHAR: Stores fixed-length strings (e.g., country codes like 'US', 'NP'). Always uses full space regardless of content.
  • VARCHAR: Stores variable-length strings (e.g., names, emails). Saves space by using only needed characters.
  • BINARY: Fixed-length binary data (e.g., fixed-size encryption key or flag in binary).
  • VARBINARY: Variable-length binary data (e.g., storing images, audio files, or PDFs).
  • TINYBLOB: Very small binary large object (up to 255 bytes) suitable for small media files like favicons.
  • TINYTEXT: Very small text string (up to 255 characters), good for short user bios or brief notes.
  • TEXT: Large text data (up to 65,535 characters), ideal for articles, product descriptions, reviews.
  • LONGTEXT: Extra-large text (up to 4GB), used for storing logs, legal documents, eBooks.
  • ENUM: List of predefined values (e.g., ENUM('small', 'medium', 'large') for clothing size).
  • BIT: Stores bit values (0 or 1). Good for binary flags like on/off, true/false.
  • TINYINT: Small integer (-128 to 127). Common for short ranges like age or ratings.
  • BOOLEAN: Alias of TINYINT (0 = FALSE, 1 = TRUE). Used for logical conditions.
  • INTEGER (INT): Whole numbers (e.g., 100, -25). Used for IDs, quantity, counters.
  • FLOAT: Approximate floating-point numbers. Ideal for measurements like height or weight.
  • DOUBLE: More precise floating-point number than FLOAT. Ideal for scientific or statistical calculations.
  • DECIMAL: Fixed-point precision number. Perfect for financial data like prices, salaries.
  • DATE: Stores only date (e.g., '2025-06-16'). Used for birthdays, registration dates.
  • DATETIME: Stores both date and time (e.g., '2025-06-16 14:30:00'). Perfect for event logs or timestamps.
